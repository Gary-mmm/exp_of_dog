# CKKS-MLaaS：基于全同态加密的猫狗分类隐私推理系统

## 设计思路

本项目实现了一套基于 CKKS 全同态加密的端到端机器学习即服务（MLaaS）隐私推理系统。在 MLaaS 场景中，用户将隐私数据上传至云端大模型进行推理时面临数据泄露风险——本系统通过密码学手段让服务器在**无法获取用户原始数据**的情况下返回正确的分类结果，实现"数据可用不可见"。

以 CIFAR-10 猫狗二分类为应用场景，系统整体框架分为四个阶段：

1. **预备实验** — 验证 CKKS 同态运算正确性、近似误差特征及噪声预算限制，为后续架构设计提供理论约束；
2. **明文预训练** — 搭建密文友好的 CNN：用平方函数替代 ReLU、平均池化替代 MaxPool，训练至 70%+ 准确率并通过 BN 融合导出权重；
3. **密码学上下文构建** — 配置 CKKS 参数（N=16384, 8 素数模数链），生成公私钥与评估密钥，实现客户端加密、服务端盲计算的密钥分发方案；
4. **端到端加密推理** — 基于 Flask 搭建服务端、Tkinter 编写 GUI 客户端，实现 im2col 编码、同态卷积、逐通道全连接、解密评估的完整链路，通信量、推理耗时、logits 精度均可度量。

系统额外包含三级安全防御协议（标签化输出、差分隐私噪声、Merkle 权重承诺），解决服务端模型权重被客户端通过求解方程组逆向的威胁。

本项目由 Deepseek V4 Pro 辅助完成。

## 设计亮点

### 1. BN 融合导出 —— 训练用 BN，推理无 BN

**问题**：BatchNorm 是训练深层网络的标配，但推理时 $y = \gamma(x-\mu)/\sigma + \beta$ 涉及除法和开方运算，无法在密文域执行。

**方案**：训练阶段正常使用 BN 加速收敛；导出时将 BN 参数**代数熔合**进前置卷积/全连接权重：

$$W' = \frac{\gamma}{\sigma} W,\quad b' = \frac{\gamma(b - \mu)}{\sigma} + \beta$$

融合后的推理图退化为纯线性+平方的组合，密文域可直接执行。此方案让训练获得 BN 的全部好处（梯度稳定、收敛快），而推理不受 BN 密文不可计算性的限制。

### 2. 逐通道 MatMul 替代 pack_vectors —— 消除 Scale 失调

**问题**：TenSEAL 的 `pack_vectors` 在合并多个密文向量时引入额外的 CKKS rescaling，导致与 bias 发生**scale 失调**——偏置相对于加权和被放大 512 倍，argmax 完全失效。

**方案**：将全连接层 $y = x W^T + b$ 分解为通道维度的累加：

$$y = \sum_{i=0}^{C-1} \text{matmul}(ch_i, W_i^T) + b$$

每个通道独立执行矩阵乘，结果通过加法累加。加法在 CKKS 中**不改变 scale**，因此该方案完全避开了 `pack_vectors` 的副作用。实测明/密文预测一致性从该优化前的 0% 提升至 **100%**。

### 3. 双模型分层验证策略

采用"先跑通简单模型，再验证复杂模型"的工程策略：

- **Tiny 模型**（纯 FC，3 层乘法）：快速验证 CKKS 序列化、网络通信、解密精度全链路正确性
- **HE-CNN 模型**（1 Conv + 2 FC，5 层乘法）：在 Tiny 模型验证通过后，引入 im2col 编码和同态卷积

这种分层验证避免了在复杂模型上直接调试密码学问题，将问题定位时间从小时级压缩到分钟级。

### 4. 完整的安全防御协议设计

针对 MLaaS 场景中"客户端通过多次查询逆向服务端模型权重"的威胁，设计了三级递进的防御体系：

| 级别 | 措施 | 安全保证 |
|------|------|---------|
| L1 | 标签化输出（$x \to \{0,1\}$，不返回 logits） | 每查询仅 1 bit 信息 |
| L2 | 差分隐私噪声（$\varepsilon$-DP） | 统计不可区分 |
| L3 | Merkle 权重承诺 + zk-CNN 协议框架 | 防权重篡改 + 零知识验证 |

### 5. CKKS 深度天花板定量分析

通过实验系统性地验证了 CKKS 在固定 $N=16384$ 下的乘法深度上限（7 层），并证明**单纯增加模数链素数无法突破该上限**——增加素数仅降低安全性而无深度收益。给出了 $N \in \{4096, 8192, 16384, 32768\}$ 的深度-通信全面对比，论证了自举是突破深度限制的唯一可行路径。

### 计算管线

```
客户端                                   服务端
──────                                   ──────
image (3×32×32)                          enc_x (im2col 编码)
    │                                        │
AvgPool2d(2) → 3×16×16               conv2d_im2col(W_conv) + b_conv
    │                                        │
per-channel im2col_encode              square()
    │                                        │
    └──── HTTP POST ────▶            Σ_i matmul(ch_i, W_fc1_i.T) + b_fc1
                                              │
                                         square()
                                              │
                                         matmul(W_fc2) + b_fc2
                                              │
    ◀─── HTTP response ────           Enc(logits_2)  或  Enc(label)
    │
decrypt → argmax → cat/dog
```

## 测试方法与使用方法

### 自动化测试

```powershell
# 快速测试 (跳过慢速 CNN 测试)
python run_tests.py --quick

# 完整测试 (含 CNN 端到端加密推理)
python run_tests.py
```

测试覆盖：

| 编号 | 测试项 | 验证内容 |
|------|--------|---------|
| 1 | CKKS 基本运算 | 加密加法、乘法解密正确性 |
| 2 | 噪声与层级 | BFV 噪声预算耗尽轮次、CKKS 层级耗尽轮次 |
| 3 | BN 融合 | 融合前后模型输出一致性 |
| 4 | Tiny 模型 | 端到端加密推理预测一致性 + logits 误差 |
| 5 | HE-CNN 模型 | 同态卷积推理预测一致性 + logits 误差 |

```

```

### 手动运行

**预备实验**：
```powershell
python test.py
```

**训练与推理**：
```powershell

# 启动服务端

# 命令行客户端

# GUI 客户端 (支持自定义图片)
```

## 未来展望

1. **GPU 推理加速**：集成 HEonGPU (CUDA FHE 库)，将同态推理从 ~5s 降至 ~0.3s，同时解锁多层卷积+自举能力；

2. **完整 zk-CNN 实现**：基于 Groth16/Plonk 实现推理正确性零知识证明，从协议设计推进到工程实现；

3. **多模型蒸馏**：利用教师模型 (ReLU CNN, 90%+ accuracy) 蒸馏为平方激活学生模型，在保持密文兼容性的前提下提升准确率；

4. **生产级优化**：公钥上下文复用（通信量降至 <1 MB）、批处理并发推理、模型量化压缩。
