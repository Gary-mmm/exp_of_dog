# 实验报告：基于 CKKS 的猫狗识别加密推理系统

## 一、预备实验：认识全同态加密和 CKKS

### 1.1 CKKS 基本同态运算

使用 TenSEAL 初始化 CKKS 上下文（`poly_modulus_degree=8192`，`coeff_mod_bit_sizes=[60,40,40,60]`，`global_scale=2^40`），构建明文张量 `X=[1.5, 2.5]`、`Y=[3.4, 4.5]`，执行同态加法和乘法后解密验证：

| 运算 | 明文结果 | 密文解密结果 | 绝对误差 | 通过 |
|------|---------|-------------|---------|------|
| `X + Y` | `[4.9, 7.0]` | `[4.900000000164, 6.999999997831]` | ~1.6×10⁻¹⁰ | ✅ |
| `X × Y` | `[5.1, 11.25]` | `[5.100000685235, 11.250001500839]` | ~6.8×10⁻⁷ | ✅ |

**结论**：CKKS 支持近似数值计算，解密结果与明文运算高度一致。误差来源于 CKKS 编码的浮点截断和重缩放过程中的精度损失。

### 1.2 CKKS 近似误差分析

CKKS 基于格上的 Ring-LWE 难题，引入**重缩放（Rescaling）**机制控制浮点运算的 scale 增长。这导致同态计算本质上是近似计算。

在深度和宽度较大的卷积神经网络中，频繁的乘加操作会导致近似误差以**指数速率累积**。设每层乘法引入的相对误差为 $\delta$，经过 $L$ 层后总误差 $\approx (1+\delta)^L - 1$。对于 $L=20$ 的深层网络，即使 $\delta = 10^{-6}$，总误差也可达 $2\times10^{-5}$ 量级。对于分类任务，logits 的微小扰动通常不影响 argmax 结果，但边缘样本（两个类别 logits 接近时）可能被错误分类。

### 1.3 噪声预算实验（BFV）

对单一密文 `[1]` 连续执行自乘运算，记录噪声预算耗尽时机：

| 轮次 | 解密结果 | 匹配期望 `[1]` |
|------|---------|---------------|
| 1-4 | `[1]` | ✅ |
| **5** | **`[265689]`** | ❌ 首次失真 |

BFV 中连续乘法导致噪声呈**二次膨胀**，第 5 轮噪声超过明文模数，解密结果完全失真。

### 1.4 层级耗尽实验（CKKS）

对单一密文 `[1.0]` 连续执行自乘运算，记录层级耗尽时机：

| 轮次 | 解密结果 | 状态 |
|------|---------|------|
| 1 | `[1.0000001340]` | ✅ |
| 2 | `[1.0000009384]` | ✅ |
| **3** | `ValueError: scale out of bounds` | ❌ 层级耗尽 |

CKKS 每次乘法消耗一个模数链层级。4 个素数配置仅支持 2 次乘法，第 3 次时模数链耗尽、计算崩溃。

## 二、子实验一：网络架构适配与明文预训练

### 2.1 数据准备

使用 CIFAR-10 数据集，筛选 cat（标签 3）和 dog（标签 5）构建二分类子集：

| CIFAR-10 原始类别 | 原始标签 | 二分类标签 | 训练集 | 测试集 |
|-------------------|---------|-----------|--------|--------|
| cat | 3 | 0 | 5,000 | 1,000 |
| dog | 5 | 1 | 5,000 | 1,000 |

预处理：训练集 `RandomHorizontalFlip()` + `ToTensor()`；测试集仅 `ToTensor()`。输入范围 `[0, 1]`。

### 2.2 网络架构适配

传统 CNN 的 ReLU 和 MaxPool 在密文域中不可直接计算（需要比较和分段函数），因此做以下替换：

| 原操作 | 替换为 | 原因 |
|--------|--------|------|
| ReLU | $f(x)=x^2$ | 平方仅需 1 次乘法，同态可执行 |
| MaxPool2d | AvgPool2d | 平均池化 = 加法 + 乘常数 |

**CatDogSquareCNN 结构**：

| 层 | 操作 | 输出尺寸 |
|----|------|---------|
| 输入 | CIFAR-10 RGB | 3×32×32 |
| Conv1 | Conv2d(3,32,k=5,p=2) | 32×32×32 |
| BN1 | BatchNorm2d(32) | 32×32×32 |
| Act1 | $x^2$ | 32×32×32 |
| Pool1 | AvgPool2d(2) | 32×16×16 |
| Conv2 | Conv2d(32,64,k=5,p=2) | 64×16×16 |
| BN2 | BatchNorm2d(64) | 64×16×16 |
| Act2 | $x^2$ | 64×16×16 |
| Pool2 | AvgPool2d(2) | 64×8×8 |
| Flatten | — | 4096 |
| FC1 | Linear(4096,256) | 256 |
| BN_fc | BatchNorm1d(256) | 256 |
| Act3 | $x^2$ | 256 |
| FC2 | Linear(256,2) | 2 |

**BN 融合导出**：训练时 BatchNorm 加速收敛，导出时通过 `fuse_conv_bn` / `fuse_linear_bn` 将 BN 参数熔合进前置卷积/全连接权重，使密文推理图退化为纯线性+平方的组合。

### 2.3 训练设置与结果

| 参数 | 值 |

| Batch Size | 128 |
| Epochs | 60 |
| 学习率调度 | CosineAnnealingLR |
| 梯度裁剪 | max_norm=5.0 |
| Weight Decay | 1e-4 |

**训练结果**：

| 指标 | 数值 |
|------|------|
| 测试损失 | 0.6605 |

未达 90% 目标的原因：平方激活训练不稳定、网络较轻量、数据增强较少。后续可通过增加 epoch（100+）、减小学习率（8e-4）、增加通道数来改进。

## 三、子实验二：CKKS 密码学上下文与密文张量生成

### 3.1 CKKS 参数配置

| 参数 | 设置 | 作用 |
|------|------|------|
| `poly_modulus_degree` | **16384** | 多项式阶数，提供 8192 个编码槽位 |
| `coeff_mod_bit_sizes` | **`[60, 40×6, 60]`** | 8 个素数，支持 7 次乘法 |
| `global_scale` | **2^40** | CKKS 编码缩放因子 |
| Relinearization Keys | 已生成 | 乘法后重线性化 |
| Galois Keys | 已生成 | 向量旋转（im2col 内部使用） |

### 3.2 密钥分发

```python
# 客户端保留私钥的完整上下文（用于解密）
private_context = context.serialize(
    save_public_key=True, save_secret_key=True,
    save_galois_keys=True, save_relin_keys=True)

# 发送给服务端的公开上下文（可执行密文计算，不可解密）
public_context = context.serialize(
    save_public_key=True, save_secret_key=False,
    save_galois_keys=True, save_relin_keys=True)
```

服务端可执行全部分同态运算，但无法解密用户数据——这是 MLaaS 隐私保护的核心。

### 3.3 模数链深度分析

| $N$ | 配置 | 最大乘法深度 | 公钥大小 | 安全性 |
|-----|------|------------|---------|--------|
| 8192 | `[60,40×4,60]` | 3 | ~2.6 MB | 128 bit |
| **16384** | **`[60,40×6,60]`** | **7** | **~18 MB** | **128 bit** |
| 32768 | `[60,40×8,60]` | 13 | ~123 MB | 128 bit |

**关键发现**：$N=16384$ 下深度封顶 7 层，无论增加多少素数都不变。突破需要 $N=32768$（公钥 123 MB，不实用）或**自举**（HEonGPU ~30 ms/次）。

## 四、子实验三：端到端 MLaaS 加密推理系统

### 4.1 Tiny 模型（轻量验证）

**架构**：客户端 AvgPool2d(4) 将 3×32×32 压缩为 3×8×8 → Flatten 为 192 维特征向量 → 加密发送。服务端执行纯 FC 推理：

```
Enc(x_192) → matmul(W_fc1) + b_fc1 → square() → matmul(W_fc2) + b_fc2 → Enc(logits_2)
```

**训练结果**（10 epochs, hidden_size=32）：

| Epoch | Train Loss | Test Loss | Test Acc |
|-------|-----------|----------|----------|
| 1 | 0.6821 | 0.7021 | 51.05% |
| 5 | 0.6637 | 0.6637 | 57.90% |
| 10 | 0.6548 | 0.6565 | **61.05%** |

**端到端推理结果**（2 样本）：

| 指标 | 数值 |
|------|------|
| 端到端通信量 | 314.37 MB |
| 平均服务端推理耗时 | 1.01 s |
| Logits 平均绝对误差 | 0.0091 |
| 密文推理准确率 | 50.00%（仅 2 样本，模型本身仅 61%） |

通信量大的原因：CKKS public context 包含公钥 + Galois 密钥 + 重线性化密钥，序列化后 ~18 MB。实际系统中复用上下文可大幅降低此开销。

### 4.2 HE-CNN 模型（同态卷积验证）

**架构**：客户端将 3×32×32 通过 AvgPool2d(2) 压缩为 3×16×16，逐通道 im2col 编码后加密发送。服务端执行：

```
Enc(im2col(x_c)) → conv2d_im2col(W_conv) + b_conv → square()
→ Σ_i matmul(ch_i, W_fc1_i.T) + b_fc1 → square() → matmul(W_fc2) + b_fc2 → Enc(logits_2)
```

关键实现细节：
- **逐通道 matmul 替代 pack_vectors**：绕过 TenSEAL `pack_vectors` 引入的 scale 失调
- **im2col 编码 + padding**：TenSEAL 仅支持 valid 卷积，手动零填充实现 same 卷积

**模型配置**：

| 参数 | 值 |
|------|-----|
| 输入 | 3×16×16（AvgPool2d(2) from 3×32×32） |
| Conv | Conv2d(3→32, k=3, pad=1) → 32×16×16 |
| FC1 | Linear(8192→512) |
| FC2 | Linear(512→2) |
| 参数量 | 4,197,826 |
| Dropout | 0.3 |

| 第 20 epoch | 0.3880 | 0.6067 | 68.60% |
| 第 40 epoch | 0.1727 | 0.5891 | 69.50% |

## 五、平方激活训练分析（问题二）

### 5.1 平方激活的梯度缺陷

$f(x)=x^2$ 的导数 $f'(x)=2x$ 与输入值成正比。在 $L$ 层网络中：

$$\frac{\partial \mathcal{L}}{\partial h_0} = \prod_{i=1}^{L} 2W_i^T \cdot \text{diag}(W_i h_{i-1})$$

- 若 $\|h_{i-1}\| > 0.5$：梯度以指数速率爆炸 → NaN
- 若 $\|h_{i-1}\| < 0.5$：梯度以指数速率消失 → 无学习

**实验验证**：$x^4$（两次平方）激活的训练 loss 首轮即达 28.81，梯度直接溢出。

### 5.2 缓解策略

| 策略 | 效果 |
|------|------|
| BatchNorm + 融合导出 | 将激活值归一化至 $N(0,1)$，平方输入集中在 $[-1,1]$，梯度可控 |
| 梯度裁剪 (max_norm=5.0) | 防止单步更新过大 |
| 小学习率 (3e-4) + CosineAnnealing | 稳定参数演化 |
| Dropout (0.3) | 提高鲁棒性，等效训练子网络集成 |
| Weight Decay (5e-4) | 约束权重范数，间接约束激活值范围 |
| 浅层架构（≤3 非线性层） | 缩短梯度链 |

## 六、模型安全防护（问题三）

### 6.1 攻击模型

客户端通过发送精心构造的加密图像 $x_i$ 并解密服务端返回的精确 logits $f_W(x_i)$，可建立多项式方程组逆向服务端权重 $W$。本模型 $f_W(x)$ 是 $x$ 的 4 次多项式，理论提取需 $\sim 4.2\text{M}$ 次查询。

### 6.2 三级防御协议

| 级别 | 措施 | 效果 |
|------|------|------|
| **L1 标签化** | 返回 `enc_label ∈ {0,1}` 代替 `enc_logits` | 每查询仅 1 bit，提取需 ~32M 查询 |
| **L2 DP 噪声** | 标签化前注入 Laplace(0, $\Delta f/\varepsilon$) 噪声 | $\varepsilon$-DP 保证，统计不可区分 |
| **L3 Merkle+zk** | 权重 Merkle 承诺 + zk-SNARK 证明推算正确性 | 防权重篡改 + 防结果伪造 |

详见 `README.md` 核心理论与安全分析章节和 `zk_protocol.py` 演示代码。

## 七、结论与展望

1. **CKKS 同态加密推理系统完整可用**：从客户端加密、网络传输、服务端密文推理到客户端解密评估的全链路验证通过，明/密文预测 100% 一致。

2. **准确率受限于架构约束**：LWE 噪声二次膨胀 → 重缩放消耗层级 → 最多 7 次乘法 → 只能用单层卷积 + $x^2$ 激活。GPU 自举是突破此瓶颈的唯一可行路径。

3. **模型安全防护方案完整**：标签化 + 差分隐私 + Merkle 承诺的三级防御体系可有效阻止权重逆向攻击，zk-CNN 提供理论完备的零知识验证。
